使用Informatica将数据导出到文件或数据库

导出时间比较

20300条数据

导出为文件：7s 导出到数据库：12s

187100条数据

导出为文件：24s 导出到数据库：3m7s（4m48s）

操作步骤

  1. 创建数据源，连接数据库，选择表

  1. 创建目标file文件（或数据表结构）

切换视图到Target

  • 导出到文件

数据库类型选择flat file

  • 导出到表

  1. 创建映射

从Sources(源数据)中把需要的源数据表拖到映射视图中；从Targets(目标数据)中把需要转换后的数据表拖到映射视图中

  1. 创建任务，配置任务数据插入方式

注：需要编辑任务确认连接是否正确，不正确需要修改，如下图所示

  1. 创建工作流，指定输出文件名及输出文件目录，拖动任务到工作流视图，并用箭头连接起来

  2. 右键执行工作流

遇到的问题

1.数据源不支持MySQL 社区版

解决方法：

1）下载社区版 ODBC Connector驱动，32位安装在 Informatica Client，64位安装在Informatica Sever

注：安装ODBC 5.2.7及更高的版本需要Visual Studio 2013或2015的支持

odbc下载地址

2）Informatica Designer配置Mysql ODBC数据源

3）Informatica Client安装目录下修改C:\Informatica\9.1.0\clients\PowerCenterClient\client\bin\powrmart.ini 配置文件

【ODBCDLL】下添加MySQL=EXTODBC.DLL

不支持 MySQL Community 版本的解决方案

2.无法创建任务

原因：映射没有保存，导致找不到映射

3.默认导出的csv文件数据格式有问题，无法处理字段中的换行符

映射中的source自定义sql，将字段中的换行符替换掉

1

SELECT T_EVENT_TYPE.EVENT_TYPE_NO, T_EVENT_TYPE.EVENT_TYPE_NAME, T_EVENT_TYPE.PARENT_NO, T_EVENT_TYPE.DISPLAY_INDEX, T_EVENT_TYPE.CREATOR, T_EVENT_TYPE.CREATION_DATE, T_EVENT_TYPE.MODIFIER, T_EVENT_TYPE.MODIFY_DATE, replace(T_EVENT_TYPE.EVENT_SAMPLE,chr(10),''), replace(T_EVENT_TYPE.EVENT_ELEMENT,chr(10),'') FROM T_EVENT_TYPE

4. 分隔符与时间戳

在Target窗口编辑Target，自定义分隔符和时间戳

  • 导出的csv文件默认分隔符是“,” 可能会与字段里的逗号冲突

修改默认分隔符

  • 修改默认时间戳格式

5.Target导出为flat file文件，文件的列名需要手动输入，非常麻烦

解决方案：将source源拖入到Target视图，修改Target表的数据库类型为flat file

https://network.informatica.com/thread/30876

6.如何将多个字段合并到一个字段中（为了解决自定义分隔符的问题）

映射中的source自定义sql，将多个字段连接为1个字段查询

1

SELECT T_EVENT_GROUP.EVENT_GROUP_NO || '|@' || T_EVENT_GROUP.OCCURS_DATE FROM T_EVENT_GROUP

8.如何增量同步数据

  • 思路：SQL语句增加过滤条件 $$v_pumpdt（初始值设为01/01/1970 00:00:01）<=MODITY_DATE< $$$SESSSTARTTIME

每次执行工作流更新 变量$$v_pumpdt的值为SESSSTARTTIME SETVARIABLE($$v_pumpdt,SESSSTARTTIME)

  • 数据量小的可以考虑全量导数据，导数据之前先将数据表清空（使用informatica会话中的pre sql功能）
