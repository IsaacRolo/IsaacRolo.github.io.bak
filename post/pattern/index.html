  • 结构型模式【类结构】

模板方法；命令；访问者；迭代器；观察者；中介者；备忘录；解释器；状态；策略；职责链

单例模式

整个系统中，对某个类只能存在一个对象实例，且该类只提供一个获取其对象实例的方法

8种写法

  • 饿汉式【静态常量】
    • 类装载时完成实例化，避免线程同步问题
    • 若不使用其实例，可能存在内存浪费
  • 饿汉式【静态代码块】
  • 懒汉式【线程不安全】
  • 懒汉式【线程安全，同步方法】
  • 懒汉式【线程安全，同步代码块】
  • 双重检查
  • 静态内部类
  • 枚举

应用场景

频繁创建销毁的对象；耗时或耗资源较多又常用的对象；工具类对象；频繁访问数据库或文件的对象

  • Hibernate中的数据源代理SessionFactory
  • Runtime【运行时数据区】类
  • 数据源Datasource

工厂模式

工厂模式的意义：

  • 将创建对象的代码提取出来，统一在工厂接口【抽象类】中管理

  • 解耦，扩展性和维护性的提升

  • 满足依赖抽象原则

简单工厂模式

  • 一个工厂负责所有对象的创建
  • 对象种类较少时可以使用
  • 当新增对象类型时，需要修改工厂内部的逻辑代码，不符合开闭原则

abstract class Car{  
    private String name;  
      
    public abstract void drive();  
      
    public String getName() {  
        return name;  
    }  
    public void setName(String name) {  
        this.name = name;  
    }  
}  
//具体产品  
class Benz extends Car{  
    public void drive(){  
        System.out.println(this.getName()+"----go-----------------------");  
    }  
}  
  
class Bmw extends Car{  
    public void drive(){  
        System.out.println(this.getName()+"----go-----------------------");  
    }  
}  
  
//简单工厂  
class Driver{  
    public static Car createCar(String car){  
        Car c = null;  
        if("Benz".equalsIgnoreCase(car))  
            c = new Benz();  
        else if("Bmw".equalsIgnoreCase(car))  
            c = new Bmw();  
        return c;  
    }  
}  
  
//main方法  
public class BossSimplyFactory {  
  
    public static void main(String[] args) throws IOException {  
        Car car = Driver.createCar("benz");  
        car.setName("benz");  
        car.drive();  
    }

工厂方法模式

  • 将工厂分为 抽象工厂+具体工厂
  • 每个具体工厂负责一种对象的创建
  • 当新增对象类型时，只需要新增一个具体工厂来继承抽象工厂，不会修改已有工厂的内部代码，符合开闭原则
  • 但是当对象种类特别多的时候，则需要创建许多的具体工厂类，若对象能划分为对象族的话可以使用抽象工厂模式来优化，以免创建过多的具体工厂类

    public void drive(){  
        System.out.println(this.getName()+"----go-----------------------");  
    }

//抽象工厂  
abstract class Driver{  
    public abstract Car createCar(String car) throws Exception;  
}  
//具体工厂（每个具体工厂负责一个具体产品）  
class BenzDriver extends Driver{  
    public Car createCar(String car) throws Exception {  
        return new Benz();  
    }  
}  
class BmwDriver extends Driver{  
    public Car createCar(String car) throws Exception {  
        return new Bmw();  
    }  
}  
  
public class Boss{ 
    public static void main(String[] args) throws Exception {  
        Driver d = new BenzDriver();  
        Car c = d.createCar("benz");   
        c.setName("benz");  
        c.drive();  
    }  
}

抽象工厂模式

  • 抽象工厂模式=简单工厂+工厂方法 【和工厂方法模式一样将工厂分为 抽象工厂+具体工厂】
  • 将工厂方法模式里的抽象工厂抽象为多个产品族
  • 目的是减少具体工厂的数量
  • 抽象工厂模式的使用前提
    • 系统中有多个产品族
    • 系统一次只可能消费其中一族产品

abstract class Driver3{ 
    
    //产品树1
    public abstract BenzCar createBenzCar(String car) throws Exception;  
    
    //产品树2
    public abstract BmwCar createBmwCar(String car) throws Exception;  
    
    //产品树3
    public abstract AudiCar createAudiCar(String car) throws Exception;  
}  
//具体工厂【产品族1】  
class SportDriver extends Driver3{  
    public BenzCar createBenzCar(String car) throws Exception {  
        return new BenzSportCar();  
    }  
    public BmwCar createBmwCar(String car) throws Exception {  
        return new BmwSportCar();  
    }  
    public AudiCar createAudiCar(String car) throws Exception {  
        return new AudiSportCar();  
    }  
}  
//【产品族2】  
class BusinessDriver extends Driver3{  
    public BenzCar createBenzCar(String car) throws Exception {  
        return new BenzBusinessCar();  
    }  
    public BmwCar createBmwCar(String car) throws Exception {  
        return new BmwBusinessCar();  
    }  
    public AudiCar createAudiCar(String car) throws Exception {  
        return new AudiBusinessCar();  
    }  
}  
  
public class BossAbstractFactory {  
  
    public static void main(String[] args) throws Exception {    
        Driver3 d = new BusinessDriver();  
        AudiCar car = d.createAudiCar("");  
        car.drive();  
    }  
}

装饰者模式

动态给对象添加一些额外的属性或行为

  • 装饰着模式和静态代理模式的区别？

两者都是对类的方法进行扩展，但装饰器模式强调的是增强自身，在被装饰之后你能够在被增强的类上使用增强后的功能。而代理模式则强调要让别人帮你去做一些本身与你业务没有太多关系的职责（记录日志、设置缓存）

  • 装饰者模式相比于继承有什么优势？

添加新的行为时，继承需要在原有方法里修改代码；而装饰者模式可以通过创建装饰者类来实现，遵循类的开闭原则

  • 使用场景：IO类

  public abstract class Sweet {
      String description = "Sweet";
  
      public String getDescription() {
          return description;
      }
  
      public abstract double cost();
  }
  public class Cake extends Sweet {
      @Override
      public String getDescription() {
          return "一个蛋糕";
      }
  
      @Override
      public double cost() {
          return 66;
      }
  }
  public abstract class Decorator extends Sweet {
      public abstract String getDescription();
  }
  public class FruitDecorator extends Decorator {
      Sweet sweet;
  
      public FruitDecorator(Sweet sweet) {
          this.sweet = sweet;
      }
  
      @Override
      public String getDescription() {
          return sweet.getDescription() + "，水果";
      }
  
      @Override
      public double cost() {
          return sweet.cost() + 10;
      }
  }
  public class CandleDecorator extends Decorator {
      Sweet sweet;
  
      public CandleDecorator(Sweet sweet) {
          this.sweet = sweet;
      }
  
      @Override
      public String getDescription() {
          return sweet.getDescription() + "，蜡烛";
      }
  
      @Override
      public double cost() {
          return sweet.cost() + 10;
      }
  }
  public static void main(String[] args) {
  
      Cake cake = new Cake();
      System.out.println(cake.getDescription() + "总共花费" + cake.cost());
  
      FruitDecorator fruitDecorator = new FruitDecorator(cake);
      System.out.println(fruitDecorator.getDescription() + "总共花费" + fruitDecorator.cost());
  
      CandleDecorator candleDecorator = new CandleDecorator(fruitDecorator);
      System.out.println(candleDecorator.getDescription() + "总共花费" + candleDecorator.cost());
  
      }

      public void run() {
          System.out.println("Thread1");

  public class StaticProxyThread {
      public static void main(String[] args) {
          //myThread 被代理对象
          MyThread myThread = new MyThread();
          //thread 代理对象，调用Thread类的run方法相当于调用MyThread类的run方法
          //    @Override
          //    public void run() {
          //        if (target != null) {
          //            target.run();
          //        }
          //    }
          Thread thread = new Thread(myThread);
          //线程处于就绪状态，并没有运行，一旦得到cpu时间片，就开始执行Thread类的run()方法
          thread.start();
  
          new Thread(new Runnable() {
              @Override
              public void run() {
                  System.out.println("Thread2");
              }
          }).start();
  
          System.out.println("Thread Main");
      }
  }

  • 静态代理的缺点

    • 代理类和被代理类都是在编译期间确定，不利于扩展

    【若被代理类新增一个方法，代理类也需要进行修改，即不利于扩展】

    • 每个代理类都只为一个接口服务，多个接口就需要创建多个代理类

    【代理类过多】

动态代理

  • 代理对象是在运行时动态创建的

  • 动态代理是通过Proxy创建代理对象，然后将接口方法“代理”给InvocationHandler完成的

  • 动态代理的本质：JDK在运行期动态创建class字节码并加载的过程

  • JDK动态代理实例【使用了JDK的反射API；被代理对象必须实现接口】

  interface PhoneFactory{
      void salePhone();
      void getPhoneName(String name);
  }
  
  //被代理类
  class NokiaPhoneFactory implements PhoneFactory{
  
      @Override
      public void salePhone() {
          System.out.println("sale Nokia");
      }
  
      @Override
      public void getPhoneName(String name) {
          System.out.println("Phone name is "+name);
      }
  
  }
  
  //反射获取代理对象[代理的是类的所有方法]
  class DynamicProxyUtil{
  
      public static Object getProxyInstance(Object object){
          return Proxy.newProxyInstance(object.getClass().getClassLoader(), object.getClass().getInterfaces(), new InvocationHandler() {
              @Override
              public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                  System.out.println("log before");
                  Object returnValue = method.invoke(object, args);
                  System.out.println("log after");
                  return returnValue;
              }
          });
      }
  }
  
  
  public class DynamicProxyTest {
  
      public static void main(String[] args) {
          PhoneFactory nokiaPhoneFactory = new NokiaPhoneFactory();
          PhoneFactory proxyInstance = (PhoneFactory) DynamicProxyUtil.getProxyInstance(nokiaPhoneFactory);
          proxyInstance.salePhone();
          proxyInstance.getPhoneName("Nokia");
      }
  }

  public class Engineer {
      // 可以被代理
      public void eat() {
          System.out.println("工程师正在吃饭");
      }
  
      // final 方法不会被生成的字类覆盖
      public final void work() {
          System.out.println("工程师正在工作");
      }
  
      // private 方法不会被生成的字类覆盖
      private void play() {
          System.out.println("this engineer is playing game");
      }
  }
  
  public class CglibProxy implements MethodInterceptor {
      private Object target;
  
      public CglibProxy(Object target) {
          this.target = target;
      }
  
      @Override
      public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
          System.out.println("###   before invocation");
          Object result = method.invoke(target, objects);
          System.out.println("###   end invocation");
          return result;
      }
  
      public static Object getProxy(Object target) {
          Enhancer enhancer = new Enhancer();
          // 设置需要代理的对象
          enhancer.setSuperclass(target.getClass());
          // 设置代理人
          enhancer.setCallback(new CglibProxy(target));
          return enhancer.create();
      }
  }
  
  public class CglibMainTest {
      public static void main(String[] args) {
          // 生成 Cglib 代理类
          Engineer engineerProxy = (Engineer) CglibProxy.getProxy(new Engineer());
          // 调用相关方法
          engineerProxy.eat();
      }
  }

代理模式应用场景

  • 用来控制真实对象访问时的权限，一般用于对象应该有不同的访问权限的时候
  • 延迟加载，用代理模式实现延迟加载的一个经典应用就在 Hibernate 框架里面。当 Hibernate 加载实体 bean 时，并不会一次性将数据库所有的数据都装载。使用代理拦截原有的 getter 方法，在真正使用对象数据时才去数据库或者其他第三方组件加载实际的数据，从而提升系统性能。
