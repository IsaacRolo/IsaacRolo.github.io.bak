鉴于SpringCloud F版本【当前最新版本Hoxton SR3】的众多组件都停止了跟新【部分还在维护】

新项目的组件选型是时候进行一波更新了

注册中心

  • Eureka
    • 消费者服务使用服务别名到注册中心中获取RPC地址
    • 底层使用HttpClient实现远程调用
    • 获取的RPC地址会缓存在JVM内存中，定时跟新
    • 搭建Eureka集群保证注册中心的可用性，多个注册中心互相注册，其他服务同时注册所有的注册中心
  • Spring Cloud Alibaba Nacos

服务调用

  • Feign【Ribbon】

    • 服务调用者使用Ribbon实现发送负载均衡请求访问其他服务

    • Ribbon负载均衡策略

    • 轮询

    • 随机

    • 根据响应时间加权

    • 过滤掉熔断的服务，选择并发最小的

    • Ribbon2种使用方式

    • RestTemplate+Ribbon

    • Feign+Ribbon

    • Feign和Ribbon的关系

    Feign是一个轻量级HTTP服务客户端，集成了Ribbon，简化了服务的调用

    • Feign原理

    • 当Spring在容器中初始化Feign Client接口的实例的时候，Spring会自动创建一个类，实现这个接口

    • 实现接口的方法，本质上是通过RestTemplate调用相应的Rest接口，将返回的结果转换成相应的类型。

    • Feign的使用

    • 启动类开启注解支持@EnableFeignClients

    • 调用者创建接口interface，添加注解@FeignClient(“server-name”)

    • Feign默认等待1s后超时

  • OpenFeign【LocalBalancer】

服务降级

  • Hystrix

    • 解决分布式系统的服务雪崩问题

    • 流程图

    • 服务限流

    • 信号量隔离

      • 使用一个原子计数器（或信号量）来记录当前有多少个线程在运行，若超过设置的最大线程个数则执行服务降级【限制了流量】
      • 缺点是不支持任务排队

    • 线程池隔离

      • 每个Command【每个方法】维护一个线程池，线程池没满则请求进入线程池，满了则执行服务降级操作
      • 缺点是产生额外开销

    • 服务熔断

    • 服务调用失败时进行服务降级，熔断服务调用，检测到服务正常后恢复链路

    • 开启熔断器后，请求次数为？次，若在指定的时间窗口期？ms内，失败率达到？%时服务熔断，快速失败并降级，？ms内再次检测是否恢复正常

    • 熔断器的打开，关闭，半开状态

    • 服务降级

    • @EnableCircuitBreaker+@HystrixCommand(fallbackMethod=“”)（配置方法的降级策略）

    • 【减少重复代码】全局默认降级方法：@DefaultProperties(defualtFallback=“”)

    • 【解耦】feign客户端调用降级：@FeignClient(name = “xxx”, fallback = xxx.class) （配置类中所有方法的降级策略）

    • 待解决疑问：实现feign客户端降级时，fallback类每个方法都需要设置降级默认返回值，冗余代码多

  • Spring Cloud Alibaba Sentinel

服务网关

  • Zuul

同步阻塞IO，每一个请求创建一个线程，使用线程池来管理线程

  • Zuul2

  • GateWay

    • 基于WebFlux，异步非阻塞IO
    • 提供API的路由功能，请求过滤功能，如熔断，限流，重试
    • 路由
    • 断言：匹配HTTP请求头中的内容，若请求与断言匹配则进行路由
    • 过滤：在请求被路由前后对请求进行修改
